Publishing on the web (GOV.UK PaaS)

We recommend using GOV.UK PaaS to get your prototype online. It’s simple and fast to deploy new versions as you work.

Once your prototype is on PaaS, other people will be able to access and try your prototype from their own computers or mobile devices.

DO NOT enter real user data in to prototypes hosted on PaaS. If your prototype stores or collects user data, talk to a security professional about appropriate security steps you must take.

A prototype deployed on PaaS is called an app - it will have a url like: your-prototype.cloudapps.digital.

You can have multiple apps running on PaaS - projects often have several so they can try different ideas out at once.

1) Sign up for an account

Request an account and your password will be sent to you within one working day - usually much less.

You'll need to be able to log in to your account each time you want to publish a new version of a prototype.

2) Install the Cloud Foundry CLI

PaaS for government is built on top Cloud Foundry and you interact with it via the terminal/commandline.

Before we can do that we need to install a CLI that will allow us to do this. Get the installer from the Cloud foundry github page and run it.

Check that it has installed correctly by typing

cf -v

You should get a messages like this, confirming the installed version:

cf version X.X.X…

For more information check thePaaS setup docs, or those of your provider.

3) Login to Cloud Foundry

Log in on the commandline by typing this

cf login -a api.cloud.service.gov.uk -u USERNAME

Where USERNAME is the email address your account was created with.

You’ll then be prompted to enter your password.

If you need to change your password you can do so with this command. After entering it you'll be prompted to enter a new password that follows the PaaS guidelines

cf passwd

4) Choose name for prototype

You need to choose a name for your app that's unique. This name will be used in the url for your prototype.

For example: govuk-payments-prototype will create an app at: govuk-payments-prototype.cloudapps.digital

Find out more about how things are organised in the PaaS

5) Push your app

For each prototype you want to put on the web you need to create what Cloud foundry calls an app.

In the folder where you've been working on your prototype, run:

cf push [NAME_OF_YOUR_APP]

Replace [NAME_OF_YOUR_APP] with the name of your prototype.

The GOVUK prototyping kit will normally take about 3-5 minutes to deploy, other kits may take longer. If you encounter any issues please contact the GOV.UK PaaS support team if there are problems - we want to make this easy for you!

Once finished you should be provided with a URL for you prototype.

6) Set a username and password

To set up credentials to protect your prototype enter the following commands on the terminal.

For the username

cf set-env [NAME_OF_YOUR_APP] USERNAME [username_here]

For the password

cf set-env [NAME_OF_YOUR_APP] PASSWORD [password_here]

You need to restart your prototype for these changes to take effect

cf restart [NAME_OF_YOUR_APP]

7) View your prototype on the web

After your work is deployed, you will be able to view it on the web by visiting https://[NAME_OF_YOUR_APP].cloudapps.digital.

If you are not sure what that is, or have forgotten what you called your app, you can find out all your deployed apps and their URLs by typing:

cf apps
